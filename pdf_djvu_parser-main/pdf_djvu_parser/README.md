# Парсер для извлечения текстовых данных из PDF и DJVU файлов

Веб-приложение для автоматического извлечения текста из документов форматов PDF и DJVU с последующим сохранением в базу данных SQLite и возможностью анализа результатов.

## 📝 Описание

Проект разработан в рамках итогового проекта по курсу **"Программирование на языке Python"** (ИТ-Академия ТУСУР).

Приложение предоставляет полный цикл работы с документами:
- Загрузка файлов (одиночная и пакетная)
- Автоматическое извлечение текста
- Сохранение метаданных и контента в базу данных
- Просмотр и скачивание результатов
- Визуализация статистики и аналитики

## ✨ Возможности

- **Парсинг PDF** — использование высокопроизводительной библиотеки PyMuPDF.
- **Парсинг DJVU** — интеграция с внешней утилитой `djvutxt` (опционально).
- **Веб-интерфейс** — удобный и минималистичный интерфейс на базе Streamlit.
- **Пакетная обработка** — загрузка и обработка нескольких файлов одновременно.
- **Хранение данных** — локальная и легковесная СУБД SQLite.
- **Экспорт результатов** — скачивание извлечённого текста в удобном формате `.txt`.
- **Аналитика** — интерактивная визуализация статистики и топ-10 документов по количеству слов.
- **Graceful Degradation** — корректная работа приложения в штатном режиме даже при отсутствии внешних системных утилит.

### Язык программирования

##  Инструкция по использованию

### Загрузка файлов
1. Перейдите на вкладку **"Загрузка"** в боковом меню.
2. Нажмите кнопку **"Upload"** и выберите один или несколько файлов в формате PDF или DJVU.
3. Нажмите кнопку **"Обработать все файлы"**.
4. Дождитесь завершения процесса. Приложение покажет информационное сообщение с количеством успешно обработанных документов и списком пропущенных файлов.

### Работа с базой данных
1. Перейдите во вкладку **"База данных"**.
2. В интерактивной таблице отобразится список всех документов со следующими метаданными: ID документа, название файла, формат, размер файла, дата обработки и количество слов.
3. Выберите конкретный документ из выпадающего списка ниже, чтобы просмотреть его извлечённый текст.
4. Вы можете **скачать** извлеченный текст в формате `.txt` или полностью **удалить** запись о файле из базы данных.

### Аналитика данных
1. Откройте вкладку **"Аналитика"**.
2. Ознакомьтесь с общей наглядной статистикой пользователя: количество документов, общее число слов и суммарный размер файлов.
3. Изучите интерактивные графики распределения документов по форматам и **Топ-10 документов** по объёму текста.

##  Опциональная поддержка DJVU

Поддержка формата DJVU является модульной. Для того чтобы приложение могло читать файлы DJVU, требуется наличие консольной утилиты `djvutxt` (компонент пакета **DjVuLibre**).

### Установка DjVuLibre в систему:

* **Windows:**
  1. Скачайте установщик с официального сайта: http://djvu.sourceforge.net
  2. Установите пакет на компьютер.
  3. Убедитесь, что путь к установленной утилите `djvutxt` добавлен в системную переменную `PATH`.
  4. Перезапустите приложение.

* **Linux (Ubuntu/Debian):**
  ```bash
  sudo apt-get update
  sudo apt-get install djvulibre-bin
  ```

* **macOS:**
  ```bash
  brew install djvu2pdf
  ```

 После установки утилиты и перезапуска приложения статус поддержки в левом нижнем углу главного экрана сменится на полную доступность всех форматов.

### Работа без DjVuLibre (Graceful Degradation)
Если утилита `djvutxt` отсутствует в системе, приложение продолжит работу в штатном режиме: файлы PDF будут обрабатываться без ограничений, а файлы DJVU будут аккуратно пропускаться с выводом подсказки для пользователя.

## Принцип работы приложения

```text
[ Загрузка файлов ] ──> [ Определение формата ]
                               │
                ┌──────────────┴──────────────┐
                ▼                             ▼
         ( Формат PDF )                ( Формат DJVU )
                │                             │
          Используется                  Проверяется PATH
       библиотека PyMuPDF             и вызывается djvutxt
                │                             │
                └──────────────┬──────────────┘
                               ▼
                    [ Подсчёт слов в тексте ]
                               │
                               ▼
               [ Сохранение метаданных и текста ]
                    (в базу данных SQLite)
                               │
                               ▼
                 [ Отображение результатов, ]
                    [ экспорт и аналитика ]
```

1. **Загрузка:** Пользователь передает файлы через встроенный загрузчик Streamlit.
2. **Маршрутизация:** Программа валидирует расширение файла.
3. **Извлечение:** В зависимости от формата вызывается либо Python-модуль `fitz`, либо системный процесс через `subprocess`.
4. **Анализ:** Полученный текстовый поток очищается, и скрипт рассчитывает статистические метрики.
5. **Синхронизация:** Данные записываются в локальную таблицу SQLite.
